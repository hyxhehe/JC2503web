二、必须分的 8 个核心步骤（照做即可）
1. 搭建环境（最关键，不能错）
安装 Node.js 24.13.0 LTS（老师指定版本）
新建项目文件夹，放入老师给的 package.json
运行：npm install（自动装 Express、Socket.io、EJS）
主文件必须叫：server.js
启动命令：npm start
2. 搭建 3 个页面结构（HTML5）
必须做 3 个页面 + 导航栏：
首页（index /root）：简单自我介绍
About 页：详细个人信息 / 兴趣 / 课程
Game 页：多人在线拼图游戏
所有页面互通导航栏
必须是合法 HTML5，要能通过验证
3. 写 CSS 样式（全站统一）
用 CSS 统一风格、颜色、布局
可使用 Bootstrap，但必须加自定义 CSS
要响应式、易读、无花哨动画
CSS 必须通过验证
4. 开发游戏后端（Node.js + Socket.io）
游戏核心规则：
4×4 棋盘
16 个唯一方块：4 形状 × 4 颜色
轮流随机发方块 → 玩家点格子放置
连成 3 个同形状 / 同颜色（横 / 竖 / 斜）就消除，返回到池，玩家得分
棋盘满 16 格 → 清空，当前玩家得 16 分大奖
支持随时加入 / 退出，轮流顺序自动调整
掉线 / 超时 1 分钟视为离开
5. 游戏前端界面（Game 页面）
输入昵称加入游戏
显示当前轮到谁
显示 4×4 棋盘
显示分数板
提示信息：轮到你、得分、消除、满盘奖励
6. 用 Socket.io 实现实时通信
必须实现的事件：
玩家加入 player-join
玩家离开 player-leave
轮到某人 turn-start
放置方块 place-block
消除得分 clear-score
棋盘满盘重置 board-full
实时同步游戏状态
7. 写 report.html（HTML 报告，600–800 词）
放在根目录，访问：localhost:8080/report.html
必须包含：
网站整体设计
游戏客户端与服务器通信（Socket.io 事件说明）
遇到的困难与解决
学到的知识点
引用 + AI 使用声明（如有）
8. 打包提交（严格按要求）
打包：HTML/CSS/JS/ 图片 /public 文件夹
不要打包：node_modules、package.json
格式：zip/rar
命名规范（老师没指定就用学号姓名）
